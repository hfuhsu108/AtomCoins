# AtomCoins 介面設計 Brief（給 Claude Design）

> **怎麼用**：整份貼給 Claude Design 當輸入即可——本檔刻意寫成自我包含，不依賴其他 docs。
> **產出目標**：5 個主畫面 ＋ 關鍵 modal 的 **HTML + Tailwind CSS** 高擬真 mockup，mobile-first，並給出桌面 RWD 版型。圖示用 **Font Awesome 6**、字體 **Noto Sans TC**。
> **回填**：設計定稿後，把採用的 design tokens 與元件結構回寫進 repo，作為實作（React + Tailwind + Dexie）的依據。

---

## 1. 產品與使用者

**原子記帳 AtomCoins**：個人自用的理財記帳 PWA。一個 App 涵蓋四件事——
1. 帳戶與收支記帳（現金、銀行、信用卡）
2. 信用卡管理（額度、出帳/繳費）
3. 台股投資追蹤（買賣、持股、損益、T+2 交割）
4. 電子發票載具匣（載具匯入、手動新增、歸帳）

**使用者**：開發者本人（具資訊背景的老師），單人單裝置為主、手機優先。重度使用、看重效率與資訊密度。

## 2. 設計原則（硬性）

- **簡潔直覺 > 花俏**。功能正常、一眼看懂最重要，不做裝飾性 UI、不過度設計。
- **資訊密度優先、留白克制**。這是天天用的工具，不是 landing page。列表為主、卡片只用於摘要。
- **mobile-first**，再 RWD 到平板/桌面。
- **金額是主角**：千分位、`NT$` 前綴、正負上色、等寬數字（tabular-nums）對齊。
- **可一鍵隱藏所有金額**（隱私）。

## 3. 視覺方向與建議 Design Tokens

淺色主題為主、深色為輔。以下是定案 token。

### 色票（淺色主題）
```
--bg:            #F6F7F9   /* App 背景 */
--surface:       #FFFFFF   /* 卡片/列表底 */
--surface-alt:   #F1F3F5   /* 次級區塊、輸入框底 */
--border:        #E6E8EB
--text:          #1B1F24   /* 主文字 */
--text-secondary:#6C7480   /* 次文字、欄位標籤 */
--text-tertiary: #9AA1AC   /* 輔助、placeholder */

--brand:         #3B5BDB   /* 主色：FAB、主要動作、導覽 active */
--brand-soft:    #EEF2FF   /* 主色淡底：active 背景、chip */

--expense:       #F08C00   /* 支出（琥珀橘） */
--income:        #3B5BDB   /* 收入（品牌藍）— 與 brand 同色 */
--warning:       #E8590C   /* 交割不足、未繳提醒（深橘，與 expense 區隔） */

--stock-buy:     #E03131   /* 台股買進（紅，依台灣慣例） */
--stock-sell:    #2F9E44   /* 台股賣出（綠，依台灣慣例） */
--positive:      #2F9E44   /* 損益為正（沿用台股綠漲紅跌的反面：正=綠） */
--negative:      #E03131   /* 損益為負 */
```

> **配色策略**：一般收支用琥珀/品牌藍，與台股紅/綠買賣標籤完全不衝突。此配色本身即對色覺障礙友善（橘 vs 藍），不另設色弱模式切換。台股買賣色仍為紅/綠，色弱使用者可透過文字標籤（買進/賣出）輔助辨識。

### 字體與字級（Noto Sans TC）
- 淨資產大數字 / 金額鍵盤輸入：**32–36px / 700**，tabular-nums
- 頁面標題：20px / 600
- 區塊標題（帳戶分組、日期分組）：15px / 600
- 內文、列表主文字：14–15px / 400–500
- 次要說明、caption：12–13px / 400
- 所有金額一律 `font-variant-numeric: tabular-nums`、千分位

### 形狀與層次
- 圓角：卡片 16px、按鈕/輸入 10px、chip/類型切換 pill 全圓、FAB 全圓（56–64px）
- 陰影克制：卡片用 `1px border + 極淡陰影`；只有 FAB、彈出面板用中等陰影
- 間距尺度：4 / 8 / 12 / 16 / 20 / 24（同 Tailwind 預設）

## 4. 版型與斷點

- **手機（< 768px）**：單欄。底部固定導覽列 **5 鍵**（高約 64px ＋ iOS safe-area）：
  `首頁 fa-house ｜ 明細 fa-receipt ｜ ＋(中央突起圓形 FAB) ｜ 報表 fa-chart-pie ｜ 設定 fa-gear`
  中央「＋」最醒目（brand 色、上浮約 16px、56–64px 圓）。
- **平板/桌面（≥ 768 / ≥ 1024px）**：改 **左側導覽欄**（icon ＋ 文字），主內容區走**雙欄**（左列表、右詳情/摘要）。
- 斷點：`sm 640 / md 768 / lg 1024 / xl 1280`。

## 5. 共用元件清單（請一併設計）

- **AccountRow**：FA6 圖示 ＋ 名稱 ＋ 餘額（信用卡顯示「已用/額度」進度、證券顯示「持股市值」＋小字交割銀行餘額）
- **TransactionRow**：類別圖示 ＋ 名稱/備註 ＋ 帳戶 ＋ 金額（上色）；變體：拆帳標記、轉帳 `A→B`、應收/應付結清狀態 badge
- **SummaryCard**：淨資產卡、本月收支卡
- **TypeSwitcher**：記帳頁頂部可橫捲 pill（支出·收入·轉帳·應收·應付·股票）
- **NumberPad**：大數字鍵盤（記帳輸入用）
- **CategoryPicker**：左右兩欄列表（左側母分類＋右側子分類），手機為 Bottom Sheet、桌面為 Dropdown
- **Chip / Tag / FilterPill**
- **AmountText**：統一金額樣式（千分位、正負色、隱藏態 `••••`）
- **EmptyState**：空清單插圖/文案
- **Toast / WarningBanner**：交割不足、Σ拆帳≠總額 等警告（**警告但允許**，不硬擋）
- **BottomNav / SideNav / FAB**

## 6. 五畫面規格（自我包含）

> 琥珀=支出、品牌藍=收入。台股買紅賣綠（台灣慣例）。所有畫面右上可放「眼睛」一鍵隱藏金額、通知鈴鐺。桌面版皆採左導覽 ＋ 雙欄。

### 畫面 1 — 首頁 Home
- 頂部 **淨資產**大數字 ＋ 眼睛切換；下方小字「較上月 ±NT$X」；點擊展開組成（現金＋銀行＋投資市值 −信用卡未繳 ＋應收 −應付）。
- **本月收支**：收入(品牌藍)/支出(琥珀)並排 ＋ 結餘。
- **帳戶列表**：依類型分組（現金 / 銀行 / 信用卡 / 證券），可摺疊；用 AccountRow。信用卡顯示已用/額度，證券顯示持股市值。

### 畫面 2 — 記帳 Add（點 ＋ 後**全螢幕**）
- 頂部 `×` / `儲存`。最上 TypeSwitcher（預設「支出」）。
- **支出/收入**：上半大字金額 ＋ 下半 NumberPad；CategoryPicker（母→子，左右兩欄列表）；帳戶（預設主帳戶）；日期（預設今天）；可展開：備註/標籤/專案；底部動作：**拆帳**（多列）、**代墊**（某列標記代墊→自動轉應收）、進階（手續費/折扣、入帳日）；頂部範本快捷。
- **轉帳**：轉出/轉入帳戶、金額、手續費、日期。
- **應收/應付**：對象、金額、帳戶、日期、備註。
- **股票**：買/賣、證券帳戶、代號（自動帶股名）、股數、成交價、券商（自動算手續費）、交割銀行、成交日（自動推 T+2）。**即時顯示交割金額與「交割銀行夠不夠」**；不足時顯示橘色警告但仍可儲存。
- 桌面：置中 modal 或右側面板。

### 畫面 3 — 明細 Transactions（三分頁：帳本 / 股票 / 發票載具）
- **帳本**：月份切換、搜尋、篩選（帳戶/類別/標籤/專案/對象/類型）、列表⇄月曆切換；依日期分組，每組 header 顯示當日收支小計；用 TransactionRow。
- **股票**：頂部投資總覽（總市值 / 未實現損益 / 報酬率）；次分頁「持股 / 交易紀錄 / 已實現損益」；持股依證券帳戶分組，顯示代號/名稱/股數/現價/平均成本/市值/未實現損益/報酬率。股票買/賣標籤依**台股慣例**（紅=買進、綠=賣出）。
- **發票載具**：從財政部載具匯入 ＋ 手動新增；發票匣（未歸帳）列出商家/金額/日期，每張可左滑「歸帳 / 略過」；切換 未歸帳 / 已歸帳。
- 桌面：雙欄（左列表、右詳情）。

### 畫面 4 — 報表 Reports（基礎版）
- 月份導航 ＋ 支出/收入切換。
- 統計摘要（本月金額 / 日均 / 筆數）。
- 分類圓餅圖（Donut）＋ 分類排名列表。
- 近 6 個月收支柱狀趨勢圖。
- 桌面：雙欄。
- **保留**：進階報表（多維度/投資專屬/淨資產趨勢/自訂報表）留給後續階段。

### 畫面 5 — 設定 Settings
- 分組列表：個人資料(Google 登入)、備份與同步（含**自動備份開關**）、帳戶管理（含信用卡出帳/繳費/額度、證券交割銀行/券商）、分類設定（母/子＋FA6 icon＋色＋排序，支出/收入兩套）、標籤/專案/對象管理、券商設定、範本管理、偏好（主題、收支配色含色弱友善、金額隱藏預設）、通知、關於。
- 桌面：雙欄。

## 7. 關鍵狀態與邊界（務必設計到）

- **金額隱藏態**：所有金額變 `••••`，眼睛圖示切 `fa-eye` / `fa-eye-slash`。
- **空狀態**：新帳戶/無交易/空發票匣的 EmptyState。
- **警告（非硬擋）**：①股票交割銀行餘額不足 ②拆帳列加總 ≠ 總額 ③信用卡額度將超——都用深橘 `#E8590C` banner/toast 提示但允許繼續。
- **應收/應付狀態**：未結清 / 部分 / 已結清 三種 badge（部分結清在實作時補充設計）。
- **損益正負**：未實現/已實現損益正綠負紅（台股慣例）。

## 9. 產出要求與約束

- 交付 **HTML + Tailwind**（utility class 為主）；CSS 變數用第 3 節的 token。
- 圖示一律 **Font Awesome 6 solid**（用 class 名稱，如 `fa-solid fa-house`）。
- 每個主畫面給 **手機版 ＋ 桌面版** 兩種版型。
- 金額格式：`NT$ 1,234,567`，正負上色，等寬數字。
- 互動以靜態 mockup 呈現即可（hover/active 態可標註），不需接真資料。

## 10. 不要做的事

- 不要做密碼鎖畫面（產品不做）。
- 不要做匯率/多幣別 UI（僅 TWD）。
- 不要做融資融券/當沖相關欄位（僅現股）。
- 不要過度裝飾：避免大圖 hero、漸層滿版、擬物陰影堆疊。
- 不要把「代墊」做成一個支出類別——代墊在資料上是「應收」，UI 上是支出列旁的「代墊」標記。
