# 03 — 情境記帳手冊（記帳邏輯的權威參考）

**通用對應**：自己花錢=`expense`（記在付款帳戶，現金/銀行/信用卡皆可）｜收到錢=`income`｜自己帳戶間搬錢/繳卡費/儲值=`transfer`（本金不算支出）｜借出=`receivable`、借入=`payable`｜一筆多類別=一筆＋多拆帳列｜買賣股票=`StockTransaction`（不算支出，損益進投資報表）｜**代墊別人=`receivable`，不是 expense**。

## A. 一般收支

- 現金買早餐 50 → `expense` 現金 splits=[{早餐,50}]。
- 薪轉入帳 → `income` 銀行 splits=[{薪資,金額}]。

## B. 信用卡

- 刷卡 1000 → `expense` 帳戶=**信用卡帳戶** tradeDate=刷卡日；卡餘額 −1000（≠從銀行扣）。
- 繳卡費 → `transfer` 銀行→信用卡（**繳款不是支出**）。
- 國外刷卡含手續費（1000＋15）→ 一筆 `expense` 拆帳 splits=[{購物,1000},{國外手續費,15}] 帳戶=信用卡。
- 分期（3 期 0 利率買 30000）→ MVP：刷卡當下 `expense 30000` 記在卡，之後每期 `transfer 10000` 銀行→卡；階段2 用 InstallmentPlan 自動拆期。

## C. 轉帳 / 提存 / 儲值卡

- 跨行轉帳 → `transfer` A→B fee=15 feeCategory=金融手續費（**手續費算支出**）。
- ATM 領現 → `transfer` 銀行→現金（跨行 fee=5）；存現金 → `transfer` 現金→銀行。
- 悠遊卡/街口 → 當一個 **帳戶**(type=cash)。儲值=`transfer` 銀行→該卡；搭車 30=`expense` 該卡 {交通,30}。

## D. 借與還

- 借朋友 1000 → `receivable` 帳戶=你出錢的帳戶 amount=1000 對象=朋友（帳戶 −1000，產生應收）。
- 全還 → 該應收加 repayment{日期,1000,收款帳戶}（帳戶 +1000，結清）；只還 600 → repayment 600，未結清 400。
- 你借 5000 → `payable` 帳戶=收到錢的帳戶 amount=5000；還 5000 → 加 repayment{5000,付款帳戶}。
- 呆帳（不還了）→ 記一筆關聯 `expense`（壞帳損失）沖掉，或保留未結清。

## E. 拆帳

- 一張單多類別（850）→ `expense` splits=[{食物,600},{日用品,250}]。
- 發票歸帳拆帳 → 同上＋`invoiceId`（流程見 `01-schema.md` §3.7）。

## F. 代墊與 AA 分帳（複合：支出＋應收）⭐ 重點

原則：**代墊別人的部分是「應收」不是「支出」**。一筆消費若混了自己花＋幫人墊，拆成兩筆（共用同帳戶/同日，用 `linkGroupId` 綁），支出統計才準、代墊才追得回。

- **刷卡 1200 的飯，自己 800、代墊同事 400**：
  - ① `expense 800` 信用卡 splits=[{餐飲,800}]
  - ② `receivable 400` 帳戶=信用卡（錢一樣從卡出）對象=同事
  - 兩筆同 `linkGroupId`。卡共欠 1200 ✓、你的真實支出只算 800 ✓。同事還錢 → ②加 repayment（入你選的帳戶）。
  - ⚠️ 別做成 `expense 1200`＋一個「代墊」類別——那會把 400 算進你的支出，錯。
- **AA：你付 3000、4 人各 750** → `expense 750`(自己)＋`receivable 2250`(代墊 3 人)。要分人追就拆 3 筆 receivable 各 750、對象各填；否則一筆 2250、對象填「聚餐 AA」。
- **UX 建議**：記消費時把某幾個拆帳列勾「代墊（誰）」，App 自動把那幾列變 `receivable`、其餘留 `expense`、自動綁 `linkGroupId`。使用者只填一次，底層仍是上述兩筆。
- **發票含代墊**：歸帳拆帳時把代墊列標記代墊 → 同樣自動拆出 receivable。

## G. 退款與折扣

- 退款（買 500 退貨）→ 記一筆關聯退款：金額入原付款帳戶、`refundOfId` 指原交易、報表上**沖減原類別支出**（不是另算收入）。刷卡退款→退到信用卡（欠款減少）。MVP 未做專屬流程前，先用 `income`(類別=退款) 頂著，日後升級成沖銷。
- 折扣/折價券（實付已折）→ 只記**實付金額**的拆帳，折扣不另記；想留紀錄寫 note。

## H. 股票

- 買台積電 1 張 @600 → `StockTransaction` buy shares=1000 price=600，fee 自動算，brokerId、settlementBankId、tradeDate、settlementDate=T+2；交割日從交割銀行扣(600000＋fee)。**不進收支統計**。
- 賣 1 張 @650 → sell，tax=floor(650000×0.003)（ETF 則 ×0.001），系統算 realizedPnl=賣出淨收入 − 1000×當時平均成本。交割日入交割銀行。**已實現損益進投資報表**。
- 零股買 50 股 → shares=50，其餘同。
- 交割銀行不夠 → 記的當下跳警告（所有未交割買賣一起算），但**仍允許記錄**。
- 交割前補錢 → 一筆 `transfer` 銀行→交割銀行，可用餘額試算會算進去。

## I. 薪資扣款 / 配息

- 薪資毛 50000、自付 2000、實領 48000 → 簡單：`income 48000`(實領) 入銀行；要追毛額：`income 50000`＋`expense 2000`(勞健保)。
- 現金股利 → 配息功能保留；要記先用 `income`(類別=股利) 入交割銀行頂著，日後做配息再正規化（會回頭調持股成本）。

## J. 進階複合

- **跨月卡費**：1/20 刷 1000（expense 記在卡、tradeDate=1/20，算 1 月支出）→ 2/20 `transfer 1000` 銀行→卡。1 月支出含這筆、2 月只是還錢不重複計。
- **代墊後對方跨行轉帳還你**：對方付手續費→你 repayment 實收 400；你吸收手續費→repayment 記實收金額即可。
- **訂金＋尾款**（訂金 2000、尾款 8000）→ 兩筆 expense 同類別、掛同 project 綁一起看。退訂退款照 G。
- **多人互相代墊又互抵**：各記 receivable/payable，最後一筆 `transfer` 結清，不追到每一分。
